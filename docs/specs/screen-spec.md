# 화면정의서 — Screen Specification

> **버전:** 1.0 · **기준:** MVP 완료 (Phase 6-a)

---

## 공통 사항

### 반응형 레이아웃 분기


| 브레이크포인트            | 레이아웃       | 네비게이션                        |
| ------------------ | ---------- | ---------------------------- |
| 모바일 (< 768px)      | 단일 컬럼      | TopBar (상단) + BottomNav (하단) |
| 태블릿 (768px–1024px) | 단일 컬럼      | IconSidebar (좌측 아이콘)         |
| 데스크탑 (≥ 1024px)    | 사이드바 + 콘텐츠 | Sidebar (좌측 전체)              |


### 공통 컴포넌트


| 컴포넌트        | 경로                                         | 역할           |
| ----------- | ------------------------------------------ | ------------ |
| TopBar      | `components/shared/layout/TopBar.tsx`      | 모바일 상단 바     |
| BottomNav   | `components/shared/layout/BottomNav.tsx`   | 모바일 하단 네비게이션 |
| IconSidebar | `components/shared/layout/IconSidebar.tsx` | 태블릿 아이콘 사이드바 |
| Sidebar     | `components/shared/layout/Sidebar.tsx`     | 데스크탑 사이드바    |
| ThemeToggle | `components/shared/ThemeToggle.tsx`        | 다크/라이트 전환    |
| PageSpinner | `components/shared/PageSpinner.tsx`        | 로딩 상태        |


### 테마

- 다크 모드 기본, 라이트 모드 지원
- 시스템 설정 연동 + 수동 전환 (ThemeToggle)
- 색상 토큰: 딥 네이비 계열(다크) / 뉴트럴 화이트 계열(라이트), 민트 포인트

## 화면 1 — 랜딩 페이지

**경로:** `/` (랜딩 전용 레이아웃, 네비게이션 없음)
**파일:** `app/landing/page.tsx`
**목적:** 신규 방문자에게 서비스를 소개하고 메인 화면 진입을 유도한다.

### 레이아웃

```
┌─────────────────────────────────┐
│  [로고]              [시작하기]  │  ← 헤더
├─────────────────────────────────┤
│                                 │
│   [Hero 섹션]                    │
│   타이틀 + 서브카피                 │
│   [지금 시작하기] CTA 버튼          │
│                                 │
├─────────────────────────────────┤
│   [기능 소개 섹션]                 │
│   타이머 / 기록 / 대시보드           │
│                                 │
└─────────────────────────────────┘
```

### 컴포넌트


| 컴포넌트    | 위치        | 설명                    |
| ------- | --------- | --------------------- |
| Hero 섹션 | 최상단       | 타이틀 + 핵심 가치 + CTA 버튼  |
| 기능 소개   | Hero 하단   | 주요 기능 3–4가지 아이콘 + 설명  |
| CTA 버튼  | Hero + 하단 | 메인 화면(`/(main)`)으로 이동 |


### 인터랙션


| 액션                 | 동작                |
| ------------------ | ----------------- |
| "시작하기" / CTA 버튼 클릭 | `/(main)` 페이지로 이동 |
| 헤더 로고 클릭           | 랜딩 페이지 최상단 스크롤    |


### 에지케이스

- 기존 사용자(localStorage에 세션/작업 데이터 있음)도 랜딩으로 진입 가능 — 별도 리다이렉트 없음

---

## 화면 2 — 메인 (타이머 + 작업 목록)

**경로:** `/(main)`
**파일:** `app/(main)/page.tsx`
**목적:** 타이머 실행과 작업 선택이 동시에 이루어지는 핵심 화면.

### 레이아웃 (데스크탑)

```
┌─────────┬──────────────────────────────────┐
│         │                                  │
│         │   [타이머 링]                      │
│ Sidebar │   [남은 시간] [phase 표시]          │
│         │   [사이클 인디케이터 ●●●○]           │
│         │   [시작/일시정지] [집중모드] [종료]     │
│         ├──────────────────────────────────┤
│         │   [작업 목록]                      │
│         │   [+ 빠른 작업 추가]                │
└─────────┴──────────────────────────────────┘
```

### 레이아웃 (모바일)

```
┌─────────────────────────────┐
│ [TopBar: 로고 + 테마 토글]  │
├─────────────────────────────┤
│                             │
│   [타이머 링]                 │
│   [남은 시간] [phase]         │
│   [사이클 인디케이터]           │
│   [컨트롤 버튼들]              │
│                             │
├─────────────────────────────┤
│   [작업 목록]                 │
│   [+ 빠른 작업 추가]           │
├─────────────────────────────┤
│ [BottomNav]                 │
└─────────────────────────────┘
```

### 컴포넌트


| 컴포넌트             | 경로                                       | 설명                       |
| ---------------- | ---------------------------------------- | ------------------------ |
| TimerDisplay     | `components/timer/TimerDisplay.tsx`      | 타이머 링 + 남은 시간 + phase 표시 |
| TimerControls    | `components/timer/TimerControls.tsx`     | 시작/일시정지/종료 버튼            |
| CycleIndicator   | `components/timer/`                      | 사이클 진행 점 표시 (●●●○)       |
| SessionBadge     | `components/timer/`                      | 현재 선택 작업 표시              |
| FocusMode        | `components/timer/FocusMode.tsx`         | 집중 모드 트리거 버튼             |
| TaskList         | `components/tasks/TaskList.tsx`          | 작업 목록                    |
| TaskItem         | `components/tasks/TaskItem.tsx`          | 개별 작업 행                  |
| TaskForm         | `components/tasks/TaskForm.tsx`          | 작업 추가/수정 모달 폼            |
| TaskQuickAddForm | `components/shared/TaskQuickAddForm.tsx` | 인라인 빠른 작업 추가             |


### 타이머 상태별 UI


| 상태       | 표시                                 |
| -------- | ---------------------------------- |
| 대기(idle) | 시작 버튼, 링 비어있음                      |
| 집중 실행 중  | 일시정지 버튼 + 종료 버튼, 링 진행, 집중 모드 버튼 활성 |
| 휴식 실행 중  | 일시정지 버튼 + 종료 버튼, 링 다른 색상           |
| 일시정지     | 재개 버튼                              |
| 사이클 완료   | 다음 단계 선택 안내                        |


### 인터랙션


| 액션       | 동작                     |
| -------- | ---------------------- |
| 작업 클릭    | 해당 작업 선택, 타이머 설정 자동 적용 |
| 드래그앤드롭   | 작업 순서 변경               |
| 작업 완료 체크 | completed 토글           |
| 작업 수정 버튼 | TaskForm 모달 열기         |
| 작업 삭제 버튼 | ConfirmDialog 표시 후 삭제  |
| 세션 종료    | 세션 기록 모달 표시            |


### 미니 타이머 (다른 페이지 이동 시)

타이머 실행 중 다른 페이지(`/dashboard`, `/journal`, `/settings`)로 이동하면 네비게이션에 미니 타이머가 뜬다. 클릭하면 메인 화면으로 돌아간다.

### 에지케이스

- 작업 없음: "작업을 추가해서 집중을 시작해보세요" 빈 상태
- 작업 미선택 상태에서 타이머 시작: 미분류 세션으로 기록

## 화면 3 — 집중 모드

**경로:** 메인 화면 오버레이 (별도 URL 없음)
**파일:** `components/timer/FocusMode.tsx`
**목적:** 타이머와 동기부여 메시지만 노출해 집중 환경을 만든다.

### 레이아웃

```
┌─────────────────────────────────┐
│ [어두운 오버레이]                   │
│                                 │
│        [타이머 링 (확대)]          │
│        [남은 시간]                │
│        [동기부여 메시지]            │
│                                 │
│        [집중 모드 종료 버튼]        │
└─────────────────────────────────┘
```

### 컴포넌트


| 컴포넌트                | 설명                                                   |
| ------------------- | ---------------------------------------------------- |
| TimerDisplay (확대)   | 240px → 300px 크기 타이머 링                               |
| MotivationalMessage | `components/shared/MotivationalMessage.tsx` - 랜덤 메시지 |
| 종료 버튼               | 집중 모드 나가기 (타이머 유지)                                   |


### 인터랙션


| 액션    | 동작                                 |
| ----- | ---------------------------------- |
| 종료 버튼 | framer-motion 0.5s 트랜지션으로 메인 화면 복귀 |
| ESC 키 | 집중 모드 종료                           |


### 에지케이스

- 타이머가 집중 모드 중 완료됨 → 알림 + 집중 모드 자동 종료 또는 유지 (UX 결정 필요)

## 화면 4 — 대시보드

**경로:** `/(main)/dashboard`
**파일:** `app/(main)/dashboard/page.tsx`
**목적:** 집중 기록을 집계해 시각화하고 성취감을 준다.

### 레이아웃

```
┌─────────┬──────────────────────────────────────┐
│         │  [일 / 주 / 월] 탭                     │
│         ├──────────────────────────────────────┤
│ Sidebar │  [오늘 집중 시간 카드]                   │
│         │  [스트릭 카드]                          │
│         ├──────────────────────────────────────┤
│         │  [카테고리별 집중 시간 차트]               │
│         │  (Recharts 막대 차트)                  │
└─────────┴──────────────────────────────────────┘
```

### 컴포넌트


| 컴포넌트            | 경로                                      | 설명                |
| --------------- | --------------------------------------- | ----------------- |
| StudyChart      | `components/dashboard/StudyChart.tsx`   | Recharts 집중 시간 차트 |
| StreakCard      | `components/dashboard/StreakCard.tsx`   | 연속 기록 표시          |
| DatePickerInput | `components/shared/DatePickerInput.tsx` | 날짜 범위 선택          |
| CategoryBadge   | `components/shared/CategoryBadge.tsx`   | 차트 범례 색상 뱃지       |


### 탭별 집계 기준


| 탭   | 집계 범위      | x축            |
| --- | ---------- | ------------- |
| 일   | 오늘         | 시간대별 (0시–23시) |
| 주   | 이번 주 (월–일) | 요일별           |
| 월   | 이번 달       | 날짜별           |


### 인터랙션


| 액션         | 동작                |
| ---------- | ----------------- |
| 탭 전환       | 집계 기간 변경, 차트 업데이트 |
| 차트 바 hover | 해당 시간 집중 시간 툴팁    |


### 에지케이스

- 데이터 없음: "아직 기록이 없어요. 타이머를 시작해볼까요?" + 메인 링크
- 삭제된 카테고리의 세션: 차트에 "기타" 또는 회색으로 표시

## 화면 5 — 저널 (세션 히스토리)

**경로:** `/(main)/journal`
**파일:** `app/(main)/journal/page.tsx`
**목적:** 저장된 세션 기록을 날짜별로 조회하고 수정/삭제한다.

### 레이아웃

```
┌─────────┬──────────────────────────────────┐
│         │  [날짜 필터]                       │
│ Sidebar ├──────────────────────────────────┤
│         │  2026년 6월 19일                   │
│         │  ┌──────────────────────────┐    │
│         │  │ [작업명] [카테고리뱃지]       │    │
│         │  │ 집중 1h 30m · 3사이클       │    │
│         │  │ "오늘 3챕터 완료"           │    │
│         │  │ [수정] [삭제]              │    │
│         │  └──────────────────────────┘    │
│         │  ┌──────────────────────────┐    │
│         │  │ ...                      │    │
│         │  └──────────────────────────┘    │
└─────────┴──────────────────────────────────┘
```

### 컴포넌트


| 컴포넌트            | 경로                                      | 설명              |
| --------------- | --------------------------------------- | --------------- |
| SessionHistory  | `components/journal/SessionHistory.tsx` | 세션 목록 + 날짜별 그룹핑 |
| SessionNote     | `components/journal/SessionNote.tsx`    | 세션 메모 표시/수정     |
| CategoryBadge   | `components/shared/CategoryBadge.tsx`   | 카테고리 색상 뱃지      |
| ConfirmDialog   | `components/shared/ConfirmDialog.tsx`   | 삭제 확인           |
| DatePickerInput | `components/shared/DatePickerInput.tsx` | 날짜 필터           |


### 인터랙션


| 액션       | 동작                         |
| -------- | -------------------------- |
| 날짜 필터 변경 | 해당 날짜 또는 범위의 세션만 표시        |
| 메모 수정 클릭 | 인라인 편집 또는 모달 편집            |
| 삭제 버튼    | ConfirmDialog → 확인 시 세션 제거 |


### 에지케이스

- 기록 없음: "아직 기록된 세션이 없어요. 첫 집중 세션을 시작해보세요!" + 메인 링크
- 메모가 없는 세션: 메모 영역 빈 상태 또는 "메모 없음" 표시

## 화면 6 — 설정

**경로:** `/(main)/settings`
**파일:** `app/(main)/settings/page.tsx`
**목적:** 타이머 기본값, 알림, 테마, 동기부여 메시지, 카테고리 관리.

### 레이아웃

```
┌─────────┬────────────────────────────────────┐
│         │  설정                               │
│ Sidebar ├────────────────────────────────────┤
│         │  ▼ 타이머 기본값                      │
│         │    집중 [25] 분                     │
│         │    휴식 [5] 분                      │
│         │    사이클 [4] 회                     │
│         ├────────────────────────────────────┤
│         │  ▼ 알림                             │
│         │    브라우저 알림  [●○]                │
│         │    소리 알람      [●○]               │
│         ├────────────────────────────────────┤
│         │  ▼ 테마                             │
│         │    [다크] [라이트] [시스템]            │
│         ├────────────────────────────────────┤
│         │  ▼ 동기부여 메시지                     │
│         │    [메시지 1]  [x]                   │
│         │    [메시지 2]  [x]                   │
│         │    [+ 메시지 추가]                    │
│         ├────────────────────────────────────┤
│         │  ▼ 카테고리                          │
│         │    ● 공부  [수정] [삭제]              │
│         │    ● 업무  [수정] [삭제]              │
│         │    [+ 카테고리 추가]                  │
│         ├────────────────────────────────────┤
│         │  ▼ 앱 정보                          │
│         │    버전 · PWA 설치 안내               │
└─────────┴────────────────────────────────────┘
```

### 컴포넌트


| 컴포넌트               | 경로                                         | 설명                        |
| ------------------ | ------------------------------------------ | ------------------------- |
| TimerSettingsGroup | `components/shared/TimerSettingsGroup.tsx` | 집중/휴식/사이클 StepperInput 3종 |
| StepperInput       | `components/shared/StepperInput.tsx`       | 숫자 증감 입력 (단위 포함)          |
| CategoryPills      | `components/shared/CategoryPills.tsx`      | 카테고리 목록 및 선택              |
| ThemeToggle        | `components/shared/ThemeToggle.tsx`        | 테마 전환 버튼                  |
| ConfirmDialog      | `components/shared/ConfirmDialog.tsx`      | 카테고리 삭제 확인                |


### 인터랙션


| 액션              | 동작                                      |
| --------------- | --------------------------------------- |
| StepperInput 변경 | 즉시 settingsStore 업데이트 + localStorage 저장 |
| 브라우저 알림 토글 on   | Notification API 권한 요청                  |
| 메시지 추가          | 입력창에서 엔터/저장 버튼 → 목록에 추가                 |
| 메시지 삭제 [x]      | 즉시 제거 (확인 없음)                           |
| 카테고리 삭제         | ConfirmDialog → 확인 시 삭제                 |
| 테마 전환           | 즉시 적용                                   |


### 에지케이스

- 동기부여 메시지 전체 삭제 시: 최소 1개 유지 규칙 (마지막 메시지 삭제 버튼 비활성)
- 브라우저 알림 권한 영구 거부 시: 토글 비활성 + "브라우저 설정에서 변경하세요" 안내
- 기본 카테고리(공부, 업무, 운동, 독서, 기타) 삭제 가능 여부: 허용 (사용자 자율)
