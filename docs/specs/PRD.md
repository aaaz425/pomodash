# PRD — Product Requirements Document

> **제품:** Pomodash · **버전:** 1.1 · **작성일:** 2026-06-19
> **현재 상태:** Phase 6-b(차별화 기능)까지 완료, Phase 7(Supabase 마이그레이션) 예정

---

## 1. Executive Summary

Pomodash는 포모도로 기법 기반 학습 집중 도구다. 작업 계획 → 집중 타이머 → 세션 기록 → 대시보드의 흐름을 하나의 앱에서 완성한다. 로그인 없이 브라우저에서 바로 쓸 수 있고, 쌓인 집중 기록이 "오늘도 해냈다"는 성취감으로 이어지게 설계했다.

---

## 2. 배경 및 문제 정의

### 문제

현대인의 집중력은 점점 짧아진다. SNS 알림, 유튜브 추천 영상, 끊임없는 메시지가 주의를 빼앗는다. 그래도 수험생은 공부해야 하고, 취업 준비생은 스펙을 쌓아야 하며, 직장인은 자기계발을 멈출 수 없다.

기존 포모도로 앱의 한계:
- 회원가입 필수 → 진입 장벽
- 기록이 보이지 않아 동기부여 부족
- 타이머만 있고 작업 관리가 없음
- 강제 전환 방식이 자율성을 저해

### 해결

**"해야 할 일이 있는 사람"이 실제로 그 일을 해내도록 돕는다.**

- 로그인 없이 바로 사용 (localStorage)
- 작업 계획 → 집중 → 기록 흐름을 한 앱에서
- 타이머는 도구, 주도권은 사용자에게 (강제 전환 없음)
- 누적 대시보드로 성취감 제공

## 3. 타겟 사용자 & 페르소나

### 주요 타겟

| 유저 그룹 | 핵심 니즈 |
|-----------|-----------|
| 수험생 (수능, 공무원, 자격증) | 과목별 공부 시간 추적, 스트릭 유지 |
| 취업 준비생 | 자기소개서, 포트폴리오 작업 시간 기록 |
| 자기계발 직장인 | 퇴근 후 사이드 프로젝트/운동 집중 |
| 집중이 어려운 사람 | 짧은 집중 사이클로 시작, 성취감 경험 |

### 페르소나 예시

**김수험 (22세, 공시 준비생)**
- 하루 10시간 공부 목표, 실제로는 SNS에 빠짐
- 과목별로 몇 시간 공부했는지 알고 싶음
- 폰 보지 않고 집중하는 시간을 눈으로 확인하고 싶음

**이직장 (29세, 직장인)**
- 퇴근 후 1–2시간 개인 프로젝트 작업
- 얼마나 했는지 매일 보고 싶음
- 앱 설치 없이 바로 쓰고 싶음

## 4. 목표 및 성공 지표

### 목표 (Phase 6-a 완료 기준)

| 목표 | 지표 | 기준 |
|------|------|------|

- 기기 간 동기화: Supabase 마이그레이션 후 로그인 유저 전환율 30%
- 네이티브 앱 전환 시 앱스토어 평점 4.5 이상

## 6. 사용자 여정 (User Journey)

### 핵심 여정: 첫 사용자

```
랜딩 페이지 방문
    → "시작하기" 클릭
    → 메인 화면 (타이머 + 작업 목록)
    → 작업 생성 (제목 + 카테고리)
    → 타이머 시작
    → 집중 모드 진입 (선택)
    → 사이클 완료 → 알림
    → 세션 종료 → 메모 작성
    → 대시보드에서 오늘 집중 시간 확인
```

### 재방문 사용자

```
메인 화면 바로 진입 (랜딩 스킵)
    → 기존 작업 선택
    → 타이머 시작
    → 다른 페이지 이동 중 미니 타이머 확인
    → 대시보드에서 누적 스트릭 확인
```

## 7. 기술 제약 및 가정

### 제약

- **로그인 없음 (MVP):** 데이터는 사용자 디바이스 브라우저에만 존재. 기기 분실이나 브라우저 초기화 시 데이터 유실
- **단일 디바이스:** 기기 간 동기화 없음 (Phase 7에서 해결)
- **오프라인 우선:** 외부 API 의존 없음 (분석 도구 제외)

### 가정

- 타겟 유저는 데스크탑 또는 모바일 브라우저를 주로 쓴다
- 집중 시간은 하루 1–8시간 범위 내
- 1인 개발로 빠른 배포 사이클 유지

### 타이머 기본값 및 허용 범위 설계 근거

기본값(집중 25분 / 휴식 5분 / 4사이클)은 Francesco Cirillo의 포모도로 기법 원작(1980년대)을 따른다. 처음 앱을 열었을 때 별도 설정 없이 바로 사용할 수 있도록 업계 표준값을 채택했다.

허용 범위는 비정상적인 입력을 방지하는 실용적 상한/하한이다. 집중 시간 상한 120분은 울트라디안 리듬 연구(Peretz Lavie, 1986)에서 인간의 자연스러운 집중 사이클을 약 90분으로 보는 점을 참고해 설정했다. 휴식 시간 하한 0분은 휴식 없이 사이클을 연속으로 돌리는 사용 방식을 허용하기 위해서다. 사이클 상한 10회는 기본값(4회) 기준 최대 2.5배로, 하루 집중 세션 한 회분으로 충분히 커버할 수 있는 범위다.
