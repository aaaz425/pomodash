## 1. 포모도로 타이머

### 설명

집중(focus)과 휴식(short-break)이 교차하는 사이클 기반 타이머. 1사이클 = 집중 → 휴식. 지정된 사이클 수가 완료되거나 사용자가 수동으로 종료하면 세션이 끝난다.

### 사용자 스토리

- As a 집중하고 싶은 사용자, I want to 타이머를 시작하고 멈출 수 있어야 한다, so that 내 페이스에 맞춰 작업할 수 있다
- As a 수험생, I want to 집중 시간과 휴식 시간을 직접 설정할 수 있어야 한다, so that 내 공부 방식에 맞게 조정할 수 있다
- As a 이동 중인 사용자, I want to 다른 페이지에 있어도 타이머가 계속 돌아가야 한다, so that 탭 전환 중에도 시간을 잃지 않는다

### Acceptance Criteria

- [ ] 시작 버튼 클릭 → 타이머가 카운트다운 시작
- [ ] 일시정지 → 카운트다운 멈춤, 재개 → 중단 시점부터 이어서 진행
- [ ] 집중 타이머 완료 → 알림 → 사용자가 다음 단계 선택 (강제 전환 없음)
- [ ] 휴식 타이머 완료 → 알림 → 다음 집중 사이클로 수동 전환
- [ ] 마지막 사이클 완료 → 세션 기록 모달 자동 표시
- [ ] 세션 종료 버튼 → 현재 진행 중이라도 즉시 세션 기록 모달 표시
- [ ] 백그라운드 탭에서도 타이머 드리프트 없음 (절대 시간 기반)
- [ ] 탭 복귀 시 즉시 남은 시간 재계산 (visibilitychange 이벤트)
- [ ] 다른 페이지 이동 시 nav 미니 타이머 표시

### 데이터 흐름

```
시작: timerStore.start() → startedAt = Date.now() 저장
Tick: useTimer 훅 → remainingSeconds = targetSeconds - elapsed 재계산
일시정지: timerStore.pause() → focusPeriod 현재 구간 종료
종료: timerStore.endSession() → focusPeriods 정규화 → Session 저장
```

### 에지케이스

| 상황 | 처리 |
|------|------|
| 타이머 실행 중 새로고침 | startedAt을 sessionStorage 또는 store에 유지 |
| focusMinutes = 0 | 설정 범위 제한으로 방지 (최소 5분) |
| 사이클 도중 작업 전환 | 타이머 유지, 새 작업 ID로 세션 계속 |

## 2. 집중 모드

### 설명

타이머 실행 중 진입하는 오버레이 화면. 어두운 배경 위에 타이머 링과 동기부여 메시지만 띄워 외부 방해 요소를 차단한다.

### 사용자 스토리

- As a 집중이 필요한 사용자, I want to 타이머 외 모든 UI를 숨길 수 있어야 한다, so that 화면 요소에 주의가 분산되지 않는다
- As a 동기부여가 필요한 사용자, I want to 명언/메시지를 볼 수 있어야 한다, so that 집중 유지에 도움이 된다

### Acceptance Criteria

- [ ] 타이머 실행 중에만 집중 모드 진입 가능
- [ ] 집중 모드 진입 시 타이머 링 + 동기부여 메시지만 표시
- [ ] 나가기 버튼으로 집중 모드 종료 (타이머는 계속 실행)
- [ ] 동기부여 메시지는 랜덤 노출 (앱 설정 메시지 목록에서)
- [ ] framer-motion 트랜지션으로 부드러운 진입/퇴장

### 에지케이스

| 상황 | 처리 |
|------|------|
| 타이머 정지 중 집중 모드 버튼 | 버튼 비활성화 또는 접근 불가 |
| 메시지 목록이 비어있음 | 기본 하드코딩 메시지 폴백 |

## 3. 작업 관리

### 설명

집중할 대상을 명시적으로 정의하는 기능. 작업은 카테고리로 분류하고, 목표 집중 시간/사이클/휴식 시간을 지정한다. 드래그앤드롭으로 순서를 조정한다.

### 사용자 스토리

- As a 계획적인 사용자, I want to 오늘 할 작업 목록을 만들 수 있어야 한다, so that 무엇을 해야 할지 명확하게 알 수 있다
- As a 학생, I want to 과목별로 카테고리를 붙일 수 있어야 한다, so that 나중에 대시보드에서 과목별 시간을 확인할 수 있다
- As a 사용자, I want to 작업 순서를 드래그로 바꿀 수 있어야 한다, so that 우선순위에 맞게 정렬할 수 있다

### Acceptance Criteria

- [ ] 작업 생성: 제목 필수, 카테고리 선택, 목표 집중/휴식/사이클 수 설정
- [ ] 작업 수정: 모든 필드 변경 가능
- [ ] 작업 삭제: 확인 다이얼로그 후 삭제
- [ ] 작업 완료 표시: 완료 토글, 목록에서 시각적 구분
- [ ] 카테고리 선택: 기본 5개(공부, 업무, 운동, 독서, 기타) + 사용자 정의
- [ ] 드래그앤드롭으로 순서 변경, 변경 즉시 localStorage 저장
- [ ] 작업 선택 시 해당 작업의 타이머 설정값 자동 적용
- [ ] 작업이 없을 때 빈 상태 안내 표시

### 데이터 흐름

```
생성: TaskForm 제출 → taskStore.addTask() → localStorage 저장
수정: TaskForm 제출 → taskStore.updateTask() → localStorage 저장
삭제: ConfirmDialog 확인 → taskStore.deleteTask() → localStorage 저장
순서: DnD 완료 → taskStore.reorder() → localStorage 저장
```

### 에지케이스

| 상황 | 처리 |
|------|------|
| 작업 제목 공백만 입력 | 폼 검증으로 방지 |
| 카테고리 삭제 시 해당 작업 | 작업의 categoryId 유지, 카테고리 없음 처리 |
| 50개 이상 작업 | 현재 제한 없음 (성능 이슈 발생 시 페이지네이션 검토) |

## 4. 세션 기록 & 메모

### 설명

세션이 끝나면 자동으로 기록되는 집중 데이터와 사용자가 작성하는 회고 메모. `/journal` 페이지에서 히스토리를 조회하고 수정/삭제한다.

### 사용자 스토리

- As a 기록을 남기고 싶은 사용자, I want to 세션이 끝나면 오늘 뭘 했는지 메모를 남길 수 있어야 한다, so that 나중에 돌아볼 수 있다
- As a 사용자, I want to 과거 세션 기록을 조회하고 잘못된 내용을 수정할 수 있어야 한다, so that 정확한 기록을 유지할 수 있다

### Acceptance Criteria

- [ ] 세션 종료 시 모달 자동 표시: 완료 사이클 수, 집중 시간, 작업명 표시
- [ ] 모달에서 회고 메모 작성 (최대 500자, 선택 사항)
- [ ] 저장 버튼 클릭 → Session 객체 localStorage에 저장
- [ ] `/journal` 페이지에서 전체 세션 히스토리 조회
- [ ] 날짜별 그룹핑, 최신순 정렬
- [ ] 세션 카드에서 메모 수정 가능
- [ ] 세션 삭제 가능 (확인 다이얼로그 후)
- [ ] 빈 히스토리 상태 표시

### 데이터 흐름

```
세션 종료 → timerStore.endSession()
    → focusPeriods 정규화 (normalizeFocusPeriods)
    → Session 객체 생성 (focusSeconds, pausedSeconds, completedCycles, ...)
    → 모달 표시 → 사용자 메모 입력
    → 저장 → sessionStore.addSession() → localStorage pomodash:sessions 업데이트
```

### 에지케이스

| 상황 | 처리 |
|------|------|
| 0초 집중 세션 (바로 종료) | 저장하되 focusSeconds = 0으로 기록 |
| 메모 저장 없이 모달 닫기 | 메모 없이 세션 저장 (note = null) |
| 500자 초과 입력 | 입력 필드에서 maxLength로 방지 |

## 5. 대시보드

### 설명

저장된 세션 데이터를 집계해 집중 패턴을 시각화한다. 일/주/월 탭 필터, 집중 시간 목표 달성률, 연속 기록(스트릭), 카테고리별 차트를 제공한다.

### 사용자 스토리

- As a 성취감을 원하는 사용자, I want to 오늘 몇 시간 공부했는지 한눈에 볼 수 있어야 한다, so that 동기부여가 유지된다
- As a 습관 형성 중인 사용자, I want to 며칠 연속으로 공부했는지 스트릭으로 확인하고 싶다, so that 연속 기록을 유지하는 게 목표가 된다
- As a 효율을 분석하는 사용자, I want to 어떤 카테고리에 시간을 많이 썼는지 차트로 보고 싶다, so that 시간 배분을 조정할 수 있다

### Acceptance Criteria

- [ ] 일/주/월 탭 전환으로 집계 기간 변경
- [ ] 선택 기간 내 총 집중 시간 표시 (focusSeconds 기준)
- [ ] 일일 목표 집중 시간 설정 및 달성률 표시
- [ ] 연속 기록(스트릭): 오늘 포함 연속 집중한 날 수
- [ ] 카테고리별 색상 구분 막대 차트 (Recharts)
- [ ] 데이터 없을 때 빈 상태 안내

### 데이터 흐름

```
대시보드 마운트 → localStorage pomodash:sessions 읽기
    → date-fns로 기간 필터링
    → focusSeconds 집계 (endedAt - startedAt 아님)
    → categoryId 기준 그룹핑 → Recharts 데이터 변환
    → 스트릭: 오늘부터 역순 연속 날짜 계산
```

### 에지케이스

| 상황 | 처리 |
|------|------|
| 세션 데이터 없음 | 빈 상태 UI + 타이머 시작 유도 |
| 삭제된 카테고리의 세션 | 차트에 "미분류"로 표시 |
| 자정 기준 스트릭 계산 | 로컬 타임존 기준 날짜 처리 (date-fns) |

## 6. 설정

### 설명

앱 전반의 기본값과 개인화 옵션을 관리한다.

### 사용자 스토리

- As a 사용자, I want to 집중/휴식/사이클 기본값을 내 스타일에 맞게 바꿀 수 있어야 한다, so that 매번 새로 설정할 필요가 없다
- As a 사용자, I want to 동기부여 메시지를 내 문구로 추가할 수 있어야 한다, so that 집중 모드에서 내가 원하는 말을 볼 수 있다

### Acceptance Criteria

**타이머 기본값**
- [ ] 집중 시간: 5–120분 범위, StepperInput으로 변경
- [ ] 휴식 시간: 0–60분 범위
- [ ] 사이클 수: 1–10회 범위
- [ ] 변경 즉시 localStorage에 저장

**알림**
- [ ] 브라우저 알림 on/off 토글 (권한 요청 포함)
- [ ] 소리 알람 on/off 토글

**테마**
- [ ] 다크/라이트/시스템 설정 3가지 옵션

**동기부여 메시지**
- [ ] 기본 메시지 목록 표시
- [ ] 사용자 메시지 추가 (입력 후 저장)
- [ ] 메시지 삭제

**카테고리**
- [ ] 카테고리 추가 (이름 + 색상)
- [ ] 카테고리 수정
- [ ] 카테고리 삭제 (확인 다이얼로그)

**앱 정보**
- [ ] 현재 버전 표시
- [ ] PWA 설치 안내

## 7. 알림

### 설명

타이머가 끝날 때 사용자에게 알리는 두 가지 채널: 브라우저 Notifications API와 소리 알람.

### 사용자 스토리

- As a 다른 탭에서 작업 중인 사용자, I want to 타이머가 끝나면 알림을 받고 싶다, so that 타이머 화면을 계속 보지 않아도 된다

### Acceptance Criteria

- [ ] 브라우저 알림: 첫 사용 시 권한 요청, 승인 시 타이머 종료 알림 전송
- [ ] 권한 거부 시 조용히 실패 (앱 오류 없음)
- [ ] 소리 알람: 설정에서 on/off, 타이머 종료 시 사운드 재생
- [ ] 브라우저/소리 알림 독립적으로 on/off 가능
- [ ] 알림은 집중 타이머 종료, 휴식 타이머 종료, 세션 완료 시점에 발생

### 에지케이스

| 상황 | 처리 |
|------|------|
| Notification API 미지원 브라우저 | 소리 알람만 동작, 브라우저 알림 옵션 숨김 |
| 권한 거부 후 재요청 | 브라우저 정책에 따름 (설정 안내 제공) |

## 8. 자유 스톱워치 모드

### 설명

고정된 사이클/시간 설정 없이 자유롭게 집중 시간을 측정하는 모드. 세션 시작 모달에서 "포모도로"와 "자유" 중 선택한다. 자유 모드는 목표치가 없어 타이머 링이 항상 가득 찬 채로 옅게 표시되고 숫자는 카운트업되며, 사이클 인디케이터는 표시하지 않는다.

### 사용자 스토리

- As a 정해진 시간 없이 몰입하고 싶은 사용자, I want to 목표 시간을 정하지 않고 집중 시간을 측정만 하고 싶다, so that 자연스럽게 끝날 때까지 집중할 수 있다

### Acceptance Criteria

- [ ] 세션 시작 모달에서 포모도로/자유 모드 선택 가능
- [ ] 자유 모드 진입 시 사이클/시간 설정 UI 대신 안내 문구 노출
- [ ] 자유 모드는 자동 완료·알림음 없이 "세션 종료" 버튼으로만 수동 종료
- [ ] 세션 기록 모달에서 자유 모드 세션은 "N/M 사이클" 대신 "자유 집중 {시간}" 문구로 표시
- [ ] 저널에서도 자유 모드 세션은 사이클 대신 집중 시간만 표시

### 에지케이스

| 상황 | 처리 |
|------|------|
| 자유 모드 세션 저장 시 completedCycles/totalCycles | 항상 0/0으로 저장 |
| 자유 모드 집중 시간 상한 clamp | 포모도로와 달리 목표치 clamp 없이 경과 전체를 focusSeconds로 인정 |
| 5초 미만 자유 모드 세션 | 포모도로와 동일하게 최소 집중 하한 미만이면 세션 기록 없이 초기화 |

---

## 9. 뱃지 수집

### 설명

세션 기록으로부터 매번 파생 계산되는 성취 뱃지. localStorage에 별도 저장하지 않고 스트릭/누적 시간/카테고리 다양성/특별 이벤트 4개 카테고리로 나눠 대시보드에 표시한다.

### 사용자 스토리

- As a 습관을 게임처럼 즐기고 싶은 사용자, I want to 조건을 채우면 뱃지를 모으고 싶다, so that 꾸준히 기록을 남길 동기가 생긴다

### Acceptance Criteria

- [ ] 대시보드에 획득한 뱃지만 기본 표시(개수 표시 포함), "뱃지 모두 보기"로 미획득 뱃지까지 카테고리별 전개
- [ ] 뱃지 hover/focus 시 이름과 설명 툴팁 표시, 미획득 뱃지는 회색조 처리
- [ ] 연속 기록(3/7/30/100일), 누적 시간(1/10/50/100시간), 카테고리 다양성(2/3/5/7종), 특별 이벤트(첫 세션/심야 집중/주말 집중/퍼펙트 세션/2시간+ 마라톤) 조건 충족 시 뱃지 획득
- [ ] 뱃지가 하나도 없을 때 빈 상태 표시

### 에지케이스

| 상황 | 처리 |
|------|------|
| 퍼펙트 세션 뱃지와 자유 모드 세션 | completedCycles/totalCycles가 0이라 자유 모드는 자동 제외 |
| 뱃지 신규 획득 시점 | 별도 알림/토스트 없음, 대시보드 재방문 시 반영 |

## 10. 집중 요약 공유 카드

### 설명

대시보드에서 선택한 기간(오늘/이번 주/이번 달/전체)의 집중 요약을 정사각형 이미지 카드로 만들어 공유하거나 다운로드하는 기능. 클라이언트 Canvas로 렌더링하며 이미지 asset 없이 텍스트/도형만으로 그린다.

### 사용자 스토리

- As a 집중 기록을 자랑하고 싶은 사용자, I want to 내 집중 요약을 이미지로 공유하고 싶다, so that SNS 등에 기록을 남길 수 있다

### Acceptance Criteria

- [ ] 대시보드 헤더의 공유 버튼 클릭 → 현재 탭 필터 기준 데이터로 공유 카드 모달 표시
- [ ] 카드에 기간 라벨, 헤드라인, 총 집중 시간, 세션 수, 연속 기록, 생성일 표시
- [ ] "다운로드" 버튼으로 PNG 저장 가능
- [ ] "공유" 버튼은 `navigator.share` 파일 공유를 지원하는 브라우저에서만 노출/동작
- [ ] 세션이 0개인 기간은 "아직 집중 기록이 없어요" 헤드라인 + '0분'으로 표시

### 에지케이스

| 상황 | 처리 |
|------|------|
| 공유 API 미지원 브라우저에서 공유 버튼 클릭 | `canShare` 재검증 후 미지원 처리, 다운로드로 대체 |
| 공유 중 사용자 취소 | 에러 없이 조용히 종료 |
| 총 집중 시간 텍스트가 카드 폭을 초과 | 폰트 크기 자동 축소(64~120px) |

## 11. 집중도/방해요소 기록 & 저널 인사이트

### 설명

세션 종료 시 집중도 자가 평점과 방해요소 태그를 선택 입력하고, 저널 페이지에서 이를 기반으로 한 인사이트와 캘린더뷰를 제공한다.

### 사용자 스토리

- As a 집중 패턴을 개선하고 싶은 사용자, I want to 세션마다 집중도와 방해요소를 남기고 싶다, so that 무엇이 나를 방해하는지 파악할 수 있다
- As a 기록을 달력으로 훑어보고 싶은 사용자, I want to 날짜별로 집중 기록이 있는 날을 한눈에 보고 싶다, so that 꾸준함을 시각적으로 확인할 수 있다

### Acceptance Criteria

- [ ] 세션 기록 모달에서 집중도(3단계: 흐트러짐/보통/몰입)와 방해요소 태그(다중 선택, 6종 프리셋)를 선택 입력, 둘 다 스킵 가능
- [ ] 저널 상세 패널에서 집중도/방해요소 사후 수정 가능
- [ ] 저널 페이지는 리스트뷰/캘린더뷰 탭 전환 제공, 인사이트 섹션은 탭과 무관하게 상단 고정 노출
- [ ] 캘린더뷰: 월별 그리드에서 집중 기록이 있는 날짜만 배지 표시, 날짜 클릭 시 해당 날 세션 목록(시간 역순) 표시
- [ ] 인사이트: 최근 14일 방해요소 빈도 상위 3개, 최근 7일 vs 이전 7일 집중도 비교, 카테고리별 평균 집중도(표본 3개 이상), 최근 30일 요일별 최저 집중도 요일

### 에지케이스

| 상황 | 처리 |
|------|------|
| 인사이트 표본 부족 | 항목별로 "아직 데이터가 부족해요" 빈 상태 표시 |
| 비교 대상 기간 데이터 없음 | "지난주 대비" 대신 "이번 주 집중도를 기록하고 있어요" 문구 |
| 캘린더뷰에서 미래 날짜/기록 없는 날 클릭 | 버튼 비활성화 |
